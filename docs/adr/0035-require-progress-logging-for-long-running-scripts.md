# ADR 0035: 長時間スクリプトに進捗ログを必須化する

## Context

AI エージェントが長めの処理を実行するために一時スクリプトや補助スクリプトを生成することがある。
特にループ内で外部 API、Web、GitHub、Slack などへ通信する処理では、通信待ち、レート制限、リトライ、無限待機により、実際には進行中でも端末出力が止まって見える。

端末が無出力のままだと、AI や利用者は正常な待機、遅い処理、hang、外部通信の停止を区別しにくい。
これは Phase / Step artifact と同じく、作業状態を外部から観測可能にするための運用契約として扱う必要がある。

## Decision

- 長めの処理を実行するためにスクリプトを生成する場合、特にループで外部通信を伴う処理では、標準出力へ進捗ログを出すことを必須にする。
- ログは処理開始、各反復または定期間隔、リトライ、完了、失敗を最低限の観測点とする。
- ログには対象件数、現在位置、処理対象 ID や URL の要約、経過時間、次の待機やリトライ予定など、再実行判断に必要な情報を含める。
- secret、token、個人情報、本文全量など、不要に機密性の高い内容はログへ出さない。
- 静かな成功を前提にするスクリプトではなく、AI と利用者が停止と進行を判別できるスクリプトを標準とする。

## Consequences

- 長時間処理の hang や外部 API 待ちを検知しやすくなる。
- Codex / Claude / Qwen / Gemini のいずれでも、生成スクリプトの観測可能性に同じ期待値を持てる。
- 出力が増えるため、ログには要約情報を優先し、機密情報や巨大なレスポンス本文を含めない判断が必要になる。
- 既存の artifact gate はファイルベースの進捗境界を扱い、このルールは実行中プロセスの標準出力による観測性を補う。
