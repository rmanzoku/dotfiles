# ADR 0007: 恒久指示はそのターンで docs と Skill へ反映する

- Status: Accepted
- Date: 2026-03-13
- Worked At: 2026-03-13 15:11 JST
- Agent Model: GPT-5 Codex

## Context

このリポジトリでは、継続的な指示を Memory ではなく git 管理ファイルへ保存する方針を採っている。
しかし、保存先の原則だけでは「どのタイミングで反映するか」が曖昧で、作業中に見つかった有用な指示や運用判断がその場で docs や Skill に反映されず、次のターンで取りこぼされることがあった。

特に、再発しやすい運用判断や複数回参照する手順は、単に覚えておくよりも、その場で人間向け文書や Skill に落としておかないと再利用性が下がる。
一方で、すべてを即時に文書化すると過剰更新になるため、どの種類の情報をどこへ反映するか、また例外をどう扱うかを明確にする必要がある。

## Decision

- 恒久性のあるユーザー指示、再発しやすい運用判断、複数回参照しそうな手順は、原則その作業ターン内で git 管理ファイルへ反映する。
- 反映先は用途ごとに使い分ける。
  - 運用ルールや判断基準は `AGENTS.md` と AI 別指示ファイル
  - 背景、経緯、継続判断は `docs/adr/`
  - 反復手順や更新フローは該当 Skill
- 共通ルールの変更は、`dot_codex/AGENTS.md`、`dot_claude/CLAUDE.md`、`dot_qwen/QWEN.md`、`dot_gemini/GEMINI.md` を同期対象として扱い、実装時判断に残さない。
- 反映不要とする例外は次に限定する。
  - 一過性の事情
  - 既存文書に重複する内容
  - ユーザーが今回は文書化不要と明示した場合
- 例外を適用した場合は、反映しなかった理由を作業結果に残す。

## Consequences

- 継続指示の保存方針が「保存先の原則」だけでなく「反映タイミング」まで明確になる。
- docs、AI 別指示ファイル、Skill のどこへ書くべきかが揃い、次のターンでの参照漏れが減る。
- 共通ルールの変更が 4 AI 向け指示ファイルへ同期されやすくなり、ツール間での運用差分が生じにくくなる。
- 一方で、恒久ルール候補を見つけた際に docs / Skill 更新の確認コストは増えるため、例外条件を限定しつつ理由を残す運用を徹底する必要がある。
