# ADR 0031: Plan 提示前の自動レビュー必須化を削除する

## 背景

Plan 提示前に別モデルで簡易レビューを必ず実行する運用は、Plan の品質担保を目的に導入した。
しかし、すべての Plan に対して `.context/` への一時保存、reviewer 起動、結果反映を要求するため、通常の作業では待ち時間と手順量が目立つようになった。

## 決定

グローバル AI 別指示ファイルから、Plan 提示前の自動レビュー必須化と固定レビュー実行テンプレートを削除する。

一方で、Phase を含む Plan の次の規則は残す。

- 各 Phase ごとに使用する Skill を明示し、使用しない Phase は `なし` と明記する。
- Phase を含む Plan は、途中確認を前提にせず完走できる粒度で提示する。

## 影響

- Plan 提示前に reviewer を自動起動する必要はなくなる。
- Plan の妥当性確認は、通常の作成者責任とし、必要な場合だけ任意でレビューを実行する。
- 任意で Codex CLI によるレビューや検証を実行する場合は、直書きの `codex exec` テンプレートではなく `codex-cli-runner` の artifact 契約を使う。
- 既存 Skill 内の「レビューゲート」参照は削除し、通常の Plan 提示へ言い換える。
