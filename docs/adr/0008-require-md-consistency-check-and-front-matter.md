# ADR 0008: Markdown 更新時は全体整合確認と Front Matter を必須にする

## Context

このリポジトリでは、AI 向け指示、運用ドキュメント、Skill 定義などを Markdown で継続的に更新している。
一方で、部分的な追記だけで作業を終えると、同一ファイル内に重複したルールや矛盾した記述、必要な派生ルールの書き漏れが残りやすい。

また、Markdown のメタデータを本文先頭の箇条書きや独自見出しで表現すると、本文とメタ情報の境界が曖昧になり、更新時の整合性確認や機械処理もしづらい。

## Decision

- `*.md` を編集した場合は、変更箇所だけでなくファイル全体を見直し、矛盾、重複、ルール漏れがないか確認し、必要なら同じターンで修正する。
- `*.md` のメタデータは本文に混在させず、Front Matter で管理する。
- このルールは AI 向け共通ルールに追加し、dotfile 更新時の実務手順として `dotfile-update` スキルにも反映する。

## Consequences

- Markdown 更新時の局所修正による整合性崩れを減らしやすくなる。
- メタデータの置き場所が統一され、本文構造と機械可読な属性を分離できる。
- 既存 Markdown に本文メタデータが残っている場合は、今後そのファイルを編集するタイミングで Front Matter への移行を検討する必要がある。
