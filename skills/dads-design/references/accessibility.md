# DADS アクセシビリティ要件

デジタル庁デザインシステム（DADS）の中核思想は **「アクセシビリティの確保を最優先事項とする」**（「誰一人取り残されない」）。企画・設計の初期段階から組み込む。出典: `/dads/guidance/accessibility/`, `/dads/webaccessibility/`, `/dads/foundations/*/accessibility/`。

## 準拠目標・参照規格
- **目標適合レベル: JIS X 8341-3:2016 適合レベル AA**。JIS X 8341-3:2016 は WCAG 2.0 と同一内容。
- WCAG 2.1 / 2.2 の達成基準 A・AA も順次取り込む方針。
- 参照: WCAG 2.2 / 2.1 / 2.0、WAI-ARIA 1.2（勧告）/1.3（WD）、ARIA Authoring Practices Guide (APG)。
- 表記基準: 情報通信アクセス協議会「対応度表記ガイドライン（2021年4月版）」。
- 法的背景: 改正障害者差別解消法（2024年4月1日施行）でウェブアクセシビリティ確保を努力義務として位置づけ。

## コントラスト比（WCAG）
| 基準 | 対象 | 比率 |
|---|---|---|
| 1.4.3 AA | テキスト一般 | 4.5:1 以上 |
| 1.4.3 AA | 大きなテキスト（24px以上、または18px太字以上） | 3:1 以上 |
| **DADS方針** | テキスト全般 | **サイズによらず 4.5:1 以上を採用** |
| 1.4.6 AAA | テキスト一般 / 大きなテキスト | 7:1 / 4.5:1 以上 |
| 1.4.11 AA | UIコンポーネント・グラフィック・フォーカスインジケーター | 3:1 以上 |

コントラスト基準点トークン（`design-tokens.md` 参照）: Gray-420(#949494)=白に3:1 / Gray-536(#767676)=白黒双方に4.5:1 / Gray-600(#666666)=黒に3:1。

適用対象外: disabled状態のUI、純粋に装飾的なテキスト、ロゴ/ブランド名、写真内の付随的文字。

## ターゲットサイズ
- 操作要素（ボタン・アイコンボタン等）: **44×44px 以上**。
- インラインのリンク等: 最小 24×24px（上下に各4pxの余白があれば確保とみなす）。

## 必ず守る原則
- 前景色と背景色を必ず両方指定する。
- **色のみで情報を伝えない**（状態・リンク・エラーは下線・アイコン・テキストを併用）。色覚多様性（P型/D型）でも判別できる配色（青系リンク等）。
- キーボードのみで全操作可能。**フォーカスインジケーターを明示**（DADSのリンクFocusは黒枠＋黄背景）。
- 画像・アイコンに適切な代替テキスト。外部/新規タブのリンクは「新しいタブを開きます」等を付与。
- 動きのあるオブジェクト（カルーセル/アニメーション）は自動再生・過度な動きに注意。
- 内容量の増減・拡大に耐えるレスポンシブな外観構造。

## 責任分界（重要）
- デザインシステムが担保できるアクセシビリティは全体の一部。**各サイトのコンテンツ独自のa11y確保は制作者側の責任**。
- コンポーネントを使えば自動的にAA達成、ではない。実装時のマークアップ（見出し階層、ラベル関連付け、テーブル構造、ARIA）を正しく行うこと。DADSサイト自体の検証でも H1欠如・テーブルマークアップ誤り・ラベル不足・キーボード非対応が不適合例として挙がっている。

## 成果物への適用時チェックリスト
- [ ] 本文テキストのコントラスト 4.5:1 以上（薄いグレー文字に注意。最小は Gray-536/#767676 相当）
- [ ] リンクは色だけでなく下線でも判別可能
- [ ] フォーカス可視（キーボード操作で輪郭が見える）
- [ ] 見出しは h1→h2→h3 の階層を飛ばさない
- [ ] 画像に alt、装飾画像は alt=""
- [ ] 操作ボタンのタップ領域 44px 以上
- [ ] 色（赤=エラー等）に加えてラベル/アイコンでも意味を伝える
